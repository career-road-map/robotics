Phase A — Johannesburg

01
Johannesburg · Month 1
Blink Machine
GPIO LED Array · Timing Logic · C on Raspberry Pi
The entry point. Wire a 16-LED array to a Raspberry Pi's GPIO pins. Write bare-metal C to drive patterns — binary counter, Knight Rider sweep, Morse code encoder. The goal is not the lights: it is learning that every robot starts with a signal going in or out of a pin.
Outcome: Understand GPIO, pull-up/pull-down resistors, timing loops, and the mental model that hardware is just bits flowing through wires. Foundation for every project that follows.
CRaspberry PiGPIOElectronics

Johannesburg · Month 1–2
Line-Following Robot
IR Sensors · PID Motor Control · Two-Wheel Chassis
Build a chassis from scratch: motors, wheels, motor driver (L298N), and a 5-sensor IR array. Implement a PID controller in Python to keep the bot centred on a black tape track. Tune Kp, Ki, Kd until the robot holds the line at speed without oscillating.
Outcome: First real-world control theory. PID is the backbone of every motor, drone, and robotic joint in the industry. You will use this for the rest of your career.

Servo Arm Controller
3-DOF Desktop Arm · Inverse Kinematics · Python GUI
Build or kit a 3-degree-of-freedom desktop robotic arm with three servo joints. Implement forward kinematics to compute end-effector position from joint angles. Then implement inverse kinematics (analytical or Jacobian-based) so the arm reaches a target XYZ coordinate. Build a Tkinter GUI with sliders for direct joint control and click-to-reach coordinate input.
Outcome: Kinematics is the mathematics of robotic movement. Every industrial arm, surgical robot, and humanoid uses IK. You now speak this language.

Autonomous Grid Navigator

Add dynamic obstacle detection: if a sensor detects a blocked cell, replan in real time.

SLAM Mapper

Pick-and-Place Vision System
YOLOv8 · Object Detection · 4-DOF Arm · Sorting Task
Mount a camera above a workspace containing coloured blocks. Run YOLOv8 (trained on custom block classes) to detect objects and predict their 2D positions. Use a calibration homography to convert pixel coordinates to robot-frame XY. Command the 4-DOF arm from Project 04 (upgraded with a gripper) to pick each block and sort it into colour-coded bins. Measure pick success rate — target ≥ 85%.
Outcome: Vision-guided manipulation. This exact pipeline runs in every Amazon fulfilment centre and every Covariant deployment. You have built a warehouse robot.

Multi-Robot Swarm Coordinator

Stanford Capstone · Aug 2029
AdaptBot — A Self-Supervised Continual Learning Framework for Physical Robots
Continual Learning · Physical Deployment · Novel Benchmark · Peer-Reviewed Target
The problem AdaptBot solves: robots trained in one environment catastrophically forget previous skills when they learn new ones (catastrophic forgetting). Current solutions either require full retraining or offline data replay — neither is viable on a deployed robot in the field.

AdaptBot is a lightweight continual learning framework for physical robots that enables online skill acquisition without catastrophic forgetting, using a combination of elastic weight consolidation (EWC) and a compressed episodic memory buffer stored directly on the robot's embedded compute. The framework integrates with ROS 2 as a drop-in middleware layer — any existing ROS 2 manipulation pipeline can use AdaptBot without rewriting their codebase.

Scope: Deploy on the arm + mobile base hardware built across Projects 04–18. Train on five sequential tasks (pick-and-place, peg-in-hole, drawer opening, cup stacking, cable routing). Measure forward transfer, backward transfer, and intransigence against three baselines (naive fine-tuning, EWC alone, PackNet). Run 2,000 trials. Publish benchmark dataset on HuggingFace. Write a 6-page paper targeting the IEEE International Conference on Robotics and Automation (ICRA) student paper track or the Conference on Robot Learning (CoRL).
Outcome: An original research contribution to a fundamental open problem in robotics, deployed on real hardware, with a public dataset and a conference submission. This is the project that gets you into Stanford. No other transfer applicant to Stanford Engineering will have this.
